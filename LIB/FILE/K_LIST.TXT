# These objects do specify a "base name", which is used when the object
# is "aware" (always true for lites)
#
# The Lites (and the One Ring) specify "physical colors", which
# over-ride the built in colors.  Note that the "One Ring" also
# has a specific check for "unknown" in which it changes its name
# to "a plain gold ring".  See "desc.c"
#
# Setting "EASY_KNOW" for LITE's means that they get "auto-identified"
# while setting it for RING's/AMULET's means that once they have been
# identified once, they can never be forgotten.  This flag must be
# specified here (not v_list.txt) for various reasons -- see "desc.c".
# Note that none of the special artifacts currently use this feature.
#
# Note that ALL artifacts are given "IGNORE_ACID/ELEC/FIRE/COLD" when
# they are created, so we do not need to specify that here.
#
# Note that the "INSTA_ART" flag is used to prevent these objects from
# being created without also turning them into artifacts.  This flag
# may or may not be used at some time in the future.
#
# Note also that there is no reason to have 6 different rings here, except
# to allow the "sub-type" to choose different colors for all of them.
