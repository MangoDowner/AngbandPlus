=== ウィザードモードとデバッグモード ===

ゲームをいじるには二つの方法があります。それはウィザードモードとデバッグ
モードです。両方ともセーブファイルに使用が記録されハイスコアリストに載ることが
できなくなります。ウィザードモードまたはデバッグモードに入ろうとすると
本当にそうするか警告を受けます。本当にそうしたい場合は「y」を入力してモードに入ります。


=== ウィザードモード ===

ウィザードモードはキャラクタが死なないようにし、
敵に与えたダメージの情報を全て詳しく表示し、アイテムが消えたとき
原因を表示します。ウィザードモードに入るにはCTRL-w を押します。


=== デバッグモード ===

デバッグモードはもう少し便利です。デバッグモードに初めて入る時は、
CTRL-a を押してから警告の問いかけに「y」を入力します。CTRL-aに
続いて入力する文字はすべてデバッグコマンドになります。コマンドは
以下の通りです。


「ESC」: デバッグモードから出ます。
「"」: アーティファクト、アイテム、モンスターのスポイラーを生成します。
「a」: 全ての悪いステータスを治します。
「b」: 場所を指定してテレポートします。
「c」: 任意のアイテムを生成します。まずアイテムのおおまかな種類を画面で選び、次に
       アイテムを選びます。
「d」: 全てを感知します。
「e」: キャラクターの能力値、経験値、所持金を自由に変更します。
「f」: アイテムを*鑑定*します。
「g」: コマンドカウント分の金か、カウントがなければ上質のアイテムを生成します。
「h」: ヒットポイントを計算しなおし、体力ランク(Life Rating)を表示します。
「i」: アイテムを鑑定します。
「j」: 任意の階に飛びます。
「k」: プレイヤーに関する情報を表示します。
「l」: 標準生成レベルがコマンドカウント以上の全てのアイテムに関して
       情報を表示します。
「m」: 周囲の地形を明らかにします。
「n」: 番号がコマンドカウントに等しいモンスターを一体召喚します。モンスターの
       番号は r_info_j.txt を見てください。
「o」: 選んだアイテムをいじります。品質、能力＋値、各種ボーナス、個数を
       変更できます。「a」を最後に押さないと変更は反映されません。同様の
       アイテムの生成をシミュレート(roll)して、現在のアイテムがどの程度平均より
       良い／悪いかも表示できます。
「p」: ショートテレポートします。
「q」: 現在のクエストを達成したことにします。
「r」: カオス守護魔神の報酬／罰を受けます。コマンドカウントでどの
       報酬を受けるか指定できます。一覧はカオス守護魔神のスポイラーを
       参照してください。リストの一番上が１に対応します。
「s」: コマンドカウント体のランダムなモンスターを召喚します。
「t」: テレポートします。
「u」: 究極マッピングです。全ての場所を明るくします。
「v」: コマンドカウント個の高級品以上のアイテムを生成します。
「w」: 全ての床を明るくします。
「x」: コマンドカウンドに等しい経験値をプラスします。カウントがなければ
       経験値を二倍にします。
「z」: 距離がコマンドカウント以内のモンスターを全て消します。
       カウントが無い場合最大視界範囲になります。
「A」: 現在のプレイヤーの属性を表示します。
「C」: コマンドカウント番目のアーティファクトを生成します。番号は
       a_info_j.txt を参照してください。
「F」: do_cmd_wiz_feature. Takes a command count.
「H」: Summons a group of similar creatures(never OOD).
「M」: Get a Mutation. Use the commmand count to specify a mutation.
「N」: Summon a pet. Must use command count to specify index # of pet.
「Z」: All the monsters on the level vanish (not killed).
「_」: Debugs the monster flow code.
「@」: Activates a python script of your choice. (For future use).

To enter a command count: type 「0」, enter the number desired, and press
enter. When prompted for a command, hold down the ctrl key and type
「a」, then immediately enter a debug command.
