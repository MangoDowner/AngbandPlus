# This file is used to initialize the "lib/raw/v_info.raw" file, which is
# used to initialize the "vault template" information for the Angband game.

# Do not modify this file unless you know exactly what you are doing,
# unless you wish to risk possible system crashes and broken savefiles.

# After modifying this file, delete the "lib/raw/v_info.raw" file.

# Note that the "spacing" in the "description" lines is very important!

# New vault types added for Zangband -TY

# File format changed for PosChengband. Width and Height are calculated. Room
# type/subtype is a text description (e.g. VAULT:LESSER). New room types for 
# non-vault regular dungeon rooms (idea from Vanilla) and also for wilderness 
# encounters. Vault choice now based on type and rarity. Support custom "letters". 
# -CTK (10/2013)

# === Understanding v_info.txt ===

# N:serial number:name
# T:type:subtype[:theme]
# W:level:max_level:rarity
# L:<letter>:<Directives>+
# M:lines giving map of vault using symbols below or specified in F: lines

# 'N' indicates the beginning of an entry. The serial number must
# increase for each new vault.

# 'T' is for type information - room type, subtype and any theme options.
# Wilderness "rooms" use terrain values for subtype (see below).

# 'W' is for "when" the room should show up - level and rarity. Rooms
# will *never* be generated below the indicated level.

# 'M' lines describe the layout of the room. Lines must be added
# with spaces to fill square dimension. The following symbols are
# allowed:

#    % - outside of the vault, where corridors may be connected
#    # - granite
#    $ - glass wall
#    X - impenetrable rock
#    Y - impenetrable glass wall
#    * - treasure or trap
#    + - secret door
#    - - secret glass door
#    ' - curtain
#    ^ - trap
#    & - monster up to 5 levels out of depth (OOD)
#    @ - monster up to 11 levels OOD
#    9 - monster up to 9 levels OOD and treasure up to 7 levels OOD
#    8 - monster up to 40 levels OOD and treasure up to 20 levels OOD
#    , - monster up to 3 levels OOD and/or treasure up to 7 levels OOD
#
#    TODO: Move these to room_pref.txt rather than hardcoding in source.

# 'L' lines give custom letters used in the 'M' map lines. You are restricted
# to at most 10 custom letters per room type, and any 'L' specifications will
# override the built-in symbols listed above.

# Version stamp (required)
