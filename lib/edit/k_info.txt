# File: k_info.txt


# This file is used to initialize the "lib/raw/k_info.raw" file,which is
# used to initialize the "object kind" information for the Angband game.

# Do not modify this file unless you know exactly what you are doing,
# unless you wish to risk possible system crashes and broken savefiles.

# After modifying this file,delete the "lib/raw/k_info.raw" file.

# Note that object zero is used for the "stack" picture (unused).

# === Understanding object.txt ===

# N: serial number : & object name~
# G: symbol : color : unidentified class
# I: tval : pval : extra
# W: rating : weight : cost
# P: (base damage) (hit bonus, damage bonus) [base armour, armour bonus]
# A: depth/rarity : depth/rarity : etc
# F: flag | flag | etc
# D: description

# 'N' indicates the beginning of an entry. The serial number must
# increase for each new item. The '&' and '~' symbols are use to
# include articles and pluralization as necessary to ensure
# grammatical correctness in object descriptions.

# 'G' is for graphics - symbol, color and unidentified object class. There are
# 15 colors,as follows:

# D - Dark Gray    w - White          s - Gray          o - Orange
# r - Red          g - Green          b - Blue          u - Brown
# W - Light Gray   v - Violet         y - Yellow        R - Light Red
# G - Light Green  B - Light Blue     U - Light Brown

# One colour,"d",and one symbol," ",are used to indicate that the colour
# and symbol are derived from the equivalent unidentified item in u_info.txt.
# For unflavoured items,both turn the object invisible.
# The unidentified object class determines which set of unidentified
# descriptions (based on o_base.txt, and u_info.txt for flavoured items) an
# object has. The unidentified graphics are used if the object type is
# unidentified, or if specified as above.

# 'I' is for basic information. The tval is for the type of item,the
# sval identifies the subtype and the pval indicates the amount of
# effect the item has,if applicable (usually a fixed value: variable pvals
# are currently hard-coded).

# The "extra" field has various meanings based on the value of tval as follows:
#   If tval is 7 (chests),it represents the capacity of the chest and whether
# it can contain gold.
#   If tval is 19 (Bow),it represents the tval of the item this object fires
# when wielded.
#   If tval is 20 (Digging),21 (Hafted),22 (Polearm) or 23 (Sword),it
# represents the skill exercised by attacking things with it. Typical values
# are 8 (close combat),22 (stab),23 (slash) and 24 (crush).
#   If tval is 30 (Boots), 31 (Gloves),32 (Helmet), 33 (Crown), 34 (Shield),
# 35 (Cloak), 36 (Soft Armour), 37 (Hard Armour), 38 (Dragon Armour), 
# 40 (Amulet) or 45 (Ring), it represents a special creation condition akin
# to the special field in e_info.txt.
#   If tval is 55 (Staff),65 (Wand) or 66 (Staff),it represents the difficulty
# with which the object is used or recharged,higher being harder.
#   If tval is 90 (Book) or 94 (Charm), it is an index into the (hard coded)
# spell table.

# 'W' is for extra information. Rating is the bonus to the level feeling given
# if this object is created then, weight is in tenth-pounds (and is unused for
# artefacts) and cost is the item's base value.

# 'P' is for power information, and lists the fixed qualities of the object as
# a weapon and as armour in roughly the same layout as appears in the game
# (any variable qualities must be included in the source code at present).
# All fields must be present here to be parsed correctly.

# 'A' is for allocation - depth and rarity,in pairs. This allows an
# item to have multiple natural depths and rarities. It is used to
# ensure that certain vital items such as food and identify scrolls
# are found throughout the dungeon. The "allocation" depth need not 
# match the depth as specified in the "W:" line.

# 'F' is for flags. These are fairly self-explanatory. As many F:
# lines may be used as are needed to specify all the flags and flags
# are separated by the '|' symbol.

# 'D' is for description. As many D: lines may be used as are needed
# to describe the object. Note that lines will need spaces at their
# ends or the beginning of the next line to prevent words from running
# together.

# Version stamp (required)
