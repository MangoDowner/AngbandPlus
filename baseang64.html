This site is for downloading Angband/64, a variant of Angband, a character-based roguelike game.
Angband/64 consists of three parts: an executable, libraries and source-files. All sources are freely available, but some parts of them are copyrighted by the original authors of Angband. Read the source for more information.

What do you need to download?

To play Angband/64, you need at least the executable and libraries. To compile it yourself, you need the sources, the libraries and the gcc c/c++ compiler. Libraries have not been included in both the binaries and the sources, so you need to download them separately. Be carefull when attempting to use the dos-sources on a unix-machine: all long filenames are lost (but the only long filenames are on unix-specific files).


What platforms does it support?

The game is known to work on MSDOS (djgpp - you need the DJGPP dos-extender, check www.delorie.com for more information) machines, win32-compatible machines (like Windows 95 & 98) using the cygwin32 version of gcc for win32 (tested on win95, should work on win98 and perhaps also on NT. Mingw32 also works. Windows-2000, WinMe, WinXP and all that other shit is untested, but should run if you can get cygwin32 to run.), and Linux (rpm-format, glibc, elf). Other machines which can run gcc will probably be able to compile the sources, the original Angband runs on almost everything, and I've added no wild graphics or such. Other compilers may work, however, I develop these sources using gcc, so you are on your own using other compilers. I know for a fact I use certain extensions (64-bit integers) of gcc, which may or may not be present in other compilers. I've had a report that compiling on OS/2 using EMX works, see the links below, and a brave soul (Bablos) has also compiled an Amiga executable (the link should be somewhere on the roguelike news pages, see below).


What is the best platform?

The best platform is probably a unix platform that fully supports signals, like Linux (www.linux.org) or FreeBSD (www.freebsd.org) to name two that run on the most popular hardware. If you chose to play DOS or Windows, I can tell you now that if the game crashes, the chances to make a succesfull emergency-save or crashdump are much smaller. There are lots of other reasons as well to avoid Windows :-)


Compressed binaries:

The DOS and Win32 binaries have been compressed with the excellent upx-program. Find more info about upx at The Ultimate Packer for Executables Homepage. The linux-executable is not packed, because that would interfere with debugging with gdb.

What can I download?


LIBRARIES : you'll need the libraries in any case, with source or binaries: download __libdist__(__lenlibdist__)

ONCE AGAIN: MAKE SURE YOU DOWNLOAD THE LIBRARIES. YOU NEED THE LIBRARIES. DON'T COMPLAIN ABOUT MISSING LIB/ DIRECTORIES IF YOU DIDN'T DOWNLOAD THE LIBRARIES. YOU CANNOT PLAY WITHOUT DOWNLOADING THE LIBRARIES TOO. IF YOU DID NOT DOWNLOAD THE LIBRARIES, YOU CANNOT PLAY. DOWNLOAD THE LIBRARIES TOO.

DOS SOURCES: download __dossrc__ (__lendossrc__).

DOS PROGRAM: download __dosbin__ (__lendosbin__). This is the DOS-386 (DJGPP) executable.


Win32 PROGRAM: download __win32bin__ (__lenwin32bin__). This is the Win32 (cygwin32, see http://sourceware.cygnus.com/cygwin) executable for Win95, Win98 (and perhaps NT - untested!).



GNU/Linux PROGAM: download __linuxbin__ (__lenlinuxbin__). This is the linux (glibc 2, elf) executable. It is in an rpm-archive. It runs on the console and under X-Windows. This depends on some X libraries, and I happen to run the latest XFree86 release, so if you use an older version and you get dependency errors, rebuild from source or try it using --force.

SOURCES : download __linuxsrc__ (__lenlinuxsrc__). These are the sources, in a bzip2 archive. If you don't have bzip2, visit http://sourceware.cygnus.com/bzip2/
